## Completed

### 1. Smart sync logic

- Added `hasNewAnalysis()` function
- Compares last analysis time with last sync time
- Skips sync when there is no new analysis (avoids unnecessary API calls)

### 2. GitHub Actions workflow

File: `.github/workflows/sonarcloud.yml`

**Triggers:**

- Every push (main, prod, dev branches)
- Every PR
- Manual trigger

**Steps:**

1. Code checkout
2. Dependencies install
3. Test coverage (optional)
4. **SonarCloud analysis**
5. Wait 90 seconds (for SonarCloud processing)
6. **Database sync trigger** (`/api/cron/syncSonarCloud`)

### 3. Coolify post-deploy hook

File: `scripts/coolify-post-deploy.sh`

To use with Coolify:

1. Go to the Coolify dashboard
2. Navigate to Application Settings > Post-Deployment Command
3. Paste the script or provide the file path

## Usage

### GitHub Actions (automatic)

```bash
# Runs automatically on every merge

```

### Manual trigger

```bash
# Click "Run workflow" in the GitHub Actions UI
# or
gh workflow run sonarcloud.yml
```

### Coolify (post-deployment)

```bash
# Coolify runs this automatically on every deployment
# No additional configuration needed
```

## Flow

```text
Code Push → GitHub Actions
    ↓
SonarCloud Analysis
    ↓
Wait 90s (processing)
    ↓
Check hasNewAnalysis()
    ↓
    ├─ No new → Skip ✓
    └─ New → Sync to DB ✓
```

## Required secrets

GitHub Repository Settings > Secrets:

- `SONAR_TOKEN` - SonarCloud API token
- `CRON_SECRET` - API endpoint auth

## Notes

- **Free plan**: No webhooks available, so GitHub Actions is used instead
- **Smart sync**: Prevents unnecessary sync operations
- **Non-blocking**: If sync fails, the build is not affected
- **Retry**: Automatically retries on the next deployment

## Manual sync

```bash
# Development
curl http://localhost:3000/api/cron/syncSonarCloud \
  -H "Authorization: Bearer YOUR_CRON_SECRET"

# Production
curl https://chrry.dev/api/cron/syncSonarCloud \
  -H "Authorization: Bearer YOUR_CRON_SECRET"
```

## Monitoring

```sql
-- Last sync time
SELECT
  project_key,
  MAX(measured_at) as last_sync,
  COUNT(*) as metric_count
FROM sonar_metrics
GROUP BY project_key;

-- Unresolved issues
SELECT severity, COUNT(*)
FROM sonar_issues
WHERE status != 'RESOLVED'
GROUP BY severity;
```
